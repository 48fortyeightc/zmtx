# 折梅听雪

一个基于 Streamlit 的交互实验应用。项目通过火山方舟 Chat Completions API 调用豆包模型，让实验参与者以“市场部员工”的身份，与 AI 助手协作完成高速吹风机宣传语/广告文案创作任务，并在完成至少 3 轮对话后保存实验数据。

## 项目类型

- Web 应用：使用 Streamlit 构建前端界面
- AI 聊天应用：通过 HTTP 请求调用大模型接口
- 实验数据采集工具：将用户 ID、对话轮次、完整对话和保存时间导出为 CSV

## 主要功能

- 展示实验说明与知情同意页面
- 为每位参与者随机生成 `USER_` 开头的用户 ID
- 提供与 AI 文案助手对话的聊天界面
- 统计用户与 AI 的对话轮次
- 限制参与者至少完成 3 轮对话后才能保存数据
- 将实验记录保存为 `hairdryer_USER_xxxxxx.csv`
- 支持清空当前会话重新开始

## 技术栈

- Python
- Streamlit
- Pandas
- Requests
- 火山方舟 Chat Completions API
- 豆包模型：`doubao-seed-2-0-lite-260428`

- Python 3.9 或更高版本
- 可访问火山方舟 API 的网络环境

## 安装依赖

```bash
pip install -r requirements.txt
```

## 启动项目

```bash
streamlit run app.py
```

启动后，Streamlit 会在终端输出本地访问地址，通常类似：

```text
http://localhost:8501
```

## 使用流程

1. 打开应用首页，阅读实验说明和知情同意内容。
2. 点击“进入实验”按钮进入聊天页面。
3. 记录页面显示的用户 ID，后续问卷可能需要填写。
4. 围绕高速吹风机宣传语创作任务，与 AI 至少完成 3 轮对话。
5. 点击保存实验数据按钮。
6. 在项目目录下查看生成的 CSV 文件。

## 数据保存格式

保存后的 CSV 文件包含以下字段：

- `user_id`：随机生成的用户 ID
- `turns`：用户输入轮次
- `dialog`：完整对话内容
- `time`：保存时间

生成文件示例：

```text
hairdryer_USER_123456.csv
```

## 注意事项

- 当前 `app.py` 中的中文文案存在编码乱码问题，运行界面可能会显示异常字符。建议后续恢复为 UTF-8 编码的正常中文文本。
- 当前 API Key 直接写在 `app.py` 中，不适合公开发布或提交到公共仓库。建议改为通过环境变量或 Streamlit secrets 配置读取。
- 应用保存 CSV 时会写入当前运行目录；如果多名参与者同时使用，需要考虑文件冲突、数据集中存储和权限问题。
- 当前项目未包含自动化测试。
